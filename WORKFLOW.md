# Как мы работаем над сайтом Latitudo

Короткая памятка для команды. Сайт состоит из **трёх частей** — важно не путать, где что меняется.

| Часть | Где живёт | Кто и как меняет |
|---|---|---|
| **Контент** (тексты, фото, телефоны городов, разделы каталога) | База данных на **проде Reg.ru** | Прямо в админке прода, вживую |
| **Код** (шаблон, CSS, вёрстка) | Папка `/local`, в Git | Локально → `git push` → деплой на сервер |
| **Движок** `/bitrix/` | Только на серверах | Не трогаем, в Git его нет |

## Рабочий процесс

### Контент → меняем на проде
Тексты, картинки, телефоны и т.п. — правим **в админке прода Reg.ru**. База одна,
изменения сразу видны всем. В Git контент не попадает (это нормально).
⚠️ В админке старого стенда Beget контент больше не правим — эти изменения на прод не попадут.

### Код → меняем через Git
1. Правим код **локально** (своя OSPanel).
2. Отправляем в GitHub:
   ```
   git add -A
   git commit -m "что сделал"
   git push
   ```
3. Выкладываем на прод — в терминале (ISPmanager → «Shell-клиент» или свой SSH):
   ```
   cd ~/www/latitudo.pro && git pull
   ```
4. Если правки шаблона не видны — сбросить кэш Битрикса
   (Админка → Настройки → Очистить кэш).

### Автономный деплой через Claude
Claude (Claude Code) выполняет весь цикл сам, **без ручного терминала**:
правит код локально → `git push` → сам заходит по SSH на прод → `git pull`.

- Доступ: SSH-алиас `regru-latitudo` (настроен в `~/.ssh/config` у Kirill), вход по ключу
  `~/.ssh/regru_latitudo`. Соединение идёт **через Beget-мост** (ProxyCommand), потому что
  Reg.ru не отвечает на SSH с зарубежных IP, а VPN у Kirill должен оставаться включённым.
- Сервер тянет GitHub своим read-only deploy-ключом (`~/.ssh/github_deploy` на сервере).
- Команда выкладки:
  ```
  ssh regru-latitudo "cd www/latitudo.pro && git pull"
  ```
- На практике: достаточно сказать Claude, что поменять, — результат смотришь на сайте.
- Старый стенд Beget тоже доступен: `ssh beget-latitudo "cd ~/latituty.beget.tech/public_html && git pull"`.

## Правила, чтобы ничего не сломать

- ⛔ **Код на сервере напрямую не редактируем** — только локально и через `git push`.
  Иначе правки затрутся при следующем `git pull`.
- ⛔ **`.htaccess` не коммитим** — он разный на локали и на серверах.
  Эталон лежит в `.htaccess.example`.
- ⛔ **Секреты (`figma.token`, пароли) в Git не кладём** — они в `.gitignore`.
- ✅ Контент — только в админке прода. Код — только через Git.
