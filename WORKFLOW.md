# Как мы работаем над сайтом Latitudo

Короткая памятка для команды. Сайт состоит из **трёх частей** — важно не путать, где что меняется.

| Часть | Где живёт | Кто и как меняет |
|---|---|---|
| **Контент** (тексты, фото, телефоны городов, разделы каталога) | База данных на **проде Reg.ru** | Прямо в админке прода, вживую |
| **Включаемые области** (`/include/*.php` — «Как мы работаем», «О компании» и т.п.) | **Файлы, в Git** — но правятся в админке | В админке прода → потом забрать в репозиторий, см. регламент ниже |
| **Код** (шаблон, CSS, вёрстка) | Папка `/local`, в Git | Локально → `git push` → деплой на сервер |
| **Движок** `/bitrix/` | Только на серверах | Не трогаем, в Git его нет |

## Рабочий процесс

### Контент → меняем на проде
Тексты, картинки, телефоны и т.п. — правим **в админке прода Reg.ru**. База одна,
изменения сразу видны всем. В Git контент не попадает (это нормально).
⚠️ В админке старого стенда Beget контент больше не правим — эти изменения на прод не попадут.

### Включаемые области → правим в админке, но забираем в Git
Блоки «Как мы работаем», «О компании», «Преимущества» и прочие из папки `/include/` —
это **файлы под git**, хотя правятся визуальным редактором админки, как обычный контент.
Отредактировали такой блок на проде — он разошёлся с репозиторием и молча ждёт.

Чем это кончается: очередной `git pull` на проде падает с «Your local changes would be
overwritten by merge» и деплой встаёт. Так уже было 2026-07-24 — и с `urlrewrite.php`,
и с `include/how-we-work.php`, где правки копились месяцами.

**Правило: поправили включаемую область в админке — сразу заберите её в репозиторий.**
Не «когда-нибудь», а в тот же день: чем дольше расхождение живёт, тем страшнее его разбирать.

Проверить, есть ли расхождения (безопасно, ничего не меняет):
```
ssh regru-latitudo "cd www/latitudo.pro && git status --short"
```
В выводе допустим только `?? yandex_*.html` — файл подтверждения Яндекса, он вне git намеренно.
Всё, что помечено `M`, — это несохранённые правки, которые надо забрать.

Забрать файл с прода в репозиторий:
```
ssh regru-latitudo "cd www/latitudo.pro && base64 -w0 include/how-we-work.php" > tmp.b64
base64 -d < tmp.b64 > include/how-we-work.php     # берём байт-в-байт, иначе конфликт вернётся
git add include/how-we-work.php && git commit -m "content: блок «Как мы работаем» с прода"
git push
```
После этого на проде `git checkout -- include/how-we-work.php && git pull`.

⚠️ **Перед `git checkout --` на проде обязательно сверьте хэши** — эта команда стирает правки
без спроса, а в них живой контент, которого больше нигде нет:
```
git rev-parse HEAD:include/how-we-work.php                                  # локально
ssh regru-latitudo "cd www/latitudo.pro && git hash-object include/how-we-work.php"
```
Совпали — отбрасывать нечего, содержимое одинаковое, `checkout` безопасен.
**Не совпали — не отбрасывать:** на проде есть правки, которых нет в вашем коммите,
их сначала надо забрать (шаг выше) и только потом повторить сверку.

Города в таких блоках не зашиваем: пишем плейсхолдеры `#REGION_NAME_DECLINE_PP#`,
`#REGION_ADDRESS#` и т.п. — их подставит по поддомену обработчик из `region.php`
(полный список — в `local/php_interface/include/region.php`).

### Код → меняем через Git
1. Правим код **локально** (своя OSPanel).
2. Отправляем в GitHub:
   ```
   git add -A
   git commit -m "что сделал"
   git push
   ```
3. Выкладываем на прод — в терминале (ISPmanager → «Shell-клиент» или свой SSH):
   ```
   cd ~/www/latitudo.pro && git pull
   ```
4. Если правки шаблона не видны — сбросить кэш Битрикса
   (Админка → Настройки → Очистить кэш).

### Автономный деплой через Claude
Claude (Claude Code) выполняет весь цикл сам, **без ручного терминала**:
правит код локально → `git push` → сам заходит по SSH на прод → `git pull`.

- Доступ: SSH-алиас `regru-latitudo` (настроен в `~/.ssh/config` у Kirill), вход по ключу
  `~/.ssh/regru_latitudo`. Соединение идёт **через Beget-мост** (ProxyCommand), потому что
  Reg.ru не отвечает на SSH с зарубежных IP, а VPN у Kirill должен оставаться включённым.
- Сервер тянет GitHub своим read-only deploy-ключом (`~/.ssh/github_deploy` на сервере).
- Команда выкладки:
  ```
  ssh regru-latitudo "cd www/latitudo.pro && git pull"
  ```
- На практике: достаточно сказать Claude, что поменять, — результат смотришь на сайте.
- Старый стенд Beget тоже доступен: `ssh beget-latitudo "cd ~/latituty.beget.tech/public_html && git pull"`.

## Правила, чтобы ничего не сломать

- ⛔ **Код на сервере напрямую не редактируем** — только локально и через `git push`.
  Иначе правки затрутся при следующем `git pull`.
- ⛔ **`.htaccess` не коммитим** — он разный на локали и на серверах.
  Эталон лежит в `.htaccess.example`.
- ⛔ **Секреты (`figma.token`, пароли) в Git не кладём** — они в `.gitignore`.
- ✅ Контент — только в админке прода. Код — только через Git.
